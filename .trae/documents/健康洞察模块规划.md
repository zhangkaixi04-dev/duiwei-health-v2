1. **内容呈现顺序符合用户价值优先级**：先展示用户最关心的数据

### 2.2 健康分计算详细规则（满分10分）

#### 计分原则
1. **针对每一天的质量计分**：不是记录了就算分，而是当天的行为/状态符合"好"的标准才算分
2. **每个模块独立计分**：睡眠、运动、饮食、排便各占10分
3. **最终健康分**：4个模块得分的平均值
4. **计算透明**：用户可以看到每个模块的具体得分和每天的达标情况

#### 详细计分表

| 模块 | 满分 | "好"的标准 | 计分规则 | 计算方式 | 示例 |
|------|------|----------|---------|---------|------|
| **睡眠模块** | 10分 | 睡眠质量达标（7-9小时 + 23点前入睡） | 按本周睡眠质量达标的天数比例计分 | (睡眠达标天数 ÷ 7) × 10 | 4天睡眠达标 → (4÷7)×10 ≈ 5.7分 |
| **运动模块** | 10分 | 运动达标（当天有运动记录 + 时长≥30分钟） | 按本周运动达标的天数比例计分 | (运动达标天数 ÷ 7) × 10 | 3天运动达标 → (3÷7)×10 ≈ 4.3分 |
| **饮食模块** | 10分 | 饮食质量达标（热量在目标±15%范围内 + 食材种类≥8种） | 按本周饮食质量达标的天数比例计分 | (饮食达标天数 ÷ 7) × 10 | 5天饮食达标 → (5÷7)×10 ≈ 7.1分 |
| **排便模块** | 10分 | 排便质量达标（Bristol分型3-5型 + 规律） | 按本周排便质量达标的天数比例计分 | (排便达标天数 ÷ 7) × 10 | 6天排便达标 → (6÷7)×10 ≈ 8.6分 |
| **健康分** | 10分 | 4个模块的平均值 | (睡眠 + 运动 + 饮食 + 排便) ÷ 4 | (5.7+4.3+7.1+8.6)÷4 = 6.4分 |

#### 各模块"好"的标准详细说明

##### 睡眠质量达标标准
- **必要条件**：当天有睡眠记录
- **充分条件**（满足其一即可）：
  - 睡眠时长在7-9小时之间 **且** 入睡时间在22:00-23:00之间
  - 或用户主观评价为"良好"或"很好"

##### 运动质量达标标准
- **必要条件**：当天有运动记录
- **充分条件**（满足其一即可）：
  - 运动时长≥30分钟
  - 或运动类型为"高强度"且时长≥20分钟

##### 饮食质量达标标准
- **必要条件**：当天有饮食记录
- **充分条件**（同时满足）：
  - 热量摄入在目标热量的85%-115%范围内
  - 食材种类≥8种

##### 排便质量达标标准
- **必要条件**：当天有排便记录
- **充分条件**（满足其一即可）：
  - Bristol分型为3、4或5型（正常形态）
  - 或用户主观评价为"正常"

#### 健康分等级说明

| 健康分范围 | 等级 | 说明 |
|------------|------|------|
| 9.0 - 10.0分 | 🌟 优秀 | 习惯非常好，继续保持！ |
| 7.0 - 8.9分 | 👍 良好 | 习惯较好，有小提升空间 |
| 5.0 - 6.9分 | ⚠️ 一般 | 习惯有较大提升空间 |
| 0.0 - 4.9分 | 📝 需要改进 | 建议从基础习惯开始培养 |

### 2.4 dietScore计算规则（满分100分）

```javascript
dietScore = min(100, Math.round((本周平均热量 / 2000) × 100))
```

| 本周平均热量 | dietScore | 说明 |
|------------|-----------|------|
| 0 kcal | 0分 | 无记录 |
| 500 kcal | 25分 | 记录较少 |
| 1000 kcal | 50分 | 记录一般 |
| 1500 kcal | 75分 | 记录较好 |
| 2000 kcal+ | 100分 | 记录完整 |

## 三、数据流转与更新机制

### 3.1 数据来源

所有数据均来自 `localStorage` 中的 `dailyLogs`，按日期存储：

```javascript
dailyLogs = {
  "2026-03-16": {
    diet: [...],           // 饮食记录
    nutrition: {...},      // 营养统计
    sleep: {...},          // 睡眠记录
    exercise: {...},       // 运动记录
    poop: {...}            // 排便记录
  },
  ...
}
```

### 3.2 响应式更新机制

1. **数据变更触发**：用户记录饮食、睡眠、运动等数据时，立即保存到 `localStorage`
2. **storage事件监听**：`WeeklyReportPanel` 组件监听 `storage` 事件
3. **自动重新计算**：数据变更后自动重新计算今日和本周洞察
4. **UI即时更新**：计算完成后立即更新UI显示
