# PRD: 合拍 AI 健康管家 (Hepai AI Health Manager)

## 1. 产品定位 (Product Positioning)
**合拍 (Hepai)** 是一款 AI 智能健康管家。
- **核心理念**: 顺时而食，因人制宜。
- **差异化优势**: 将中医九大体质理论与现代营养学深度融合，提供“比亲妈更懂你”的个性化健康建议。
- **交互形态**: 对话式 UI (Chat UI)，模拟微信聊天体验。

## 2. 核心功能 (Core Features)

### 2.1. 智能饮食分析 (Smart Diet Analysis)
**功能描述**: 用户输入食物（文字/语音/图片），AI 自动分析热量、营养并结合体质给出建议。

#### 2.1.1. 分析逻辑 (Analysis Logic)
系统采用 **“双层漏斗”** 分析机制：
1.  **第一层：本地规则引擎 (Local Rules)** - *高频、离线、快速*
    -   **数据源**: 本地 500+ 种常见食物营养库 (`src/data/foodLibrary.js`).
    -   **识别**: 自动提取食物名称与份量（如“200g 米饭”）。
    -   **定性**: 判断食物属性（寒凉/温热/辛辣/发物/油腻/甜腻）。
2.  **第二层：大模型推理 (LLM)** - *长尾、复杂、深度*
    -   调用 Doubao/Zhipu API 对未收录食物或复杂菜品进行分析。

#### 2.1.2. 体质匹配矩阵 (Constitution Matrix)
系统根据用户体质对同一食物给出不同反馈：

| 体质类型 | 核心禁忌 (Key Avoidance) | 推荐方向 (Recommendation) | 示例 (Example) |
| :--- | :--- | :--- | :--- |
| **阳虚质** | 忌生冷、寒凉、冰饮、水果过量 | 宜温补、姜茶、热食 | 吃冰激凌 -> 警告：伤阳气，建议喝姜茶补救 |
| **湿热质** | 忌辛辣、酒、油炸、热带水果(芒果/榴莲) | 宜清热、祛湿、绿豆、冬瓜 | 吃麻辣火锅 -> 警告：助湿生热，建议喝金银花茶 |
| **痰湿质** | 忌肥甘厚味、甜食、奶茶、滋腻 | 宜陈皮、普洱、清淡、运动 | 喝奶茶 -> 警告：生痰之源，建议快走30分钟 |
| **气虚质** | 忌耗气食物(萝卜/山楂)、生冷 | 宜甘温补气(鸡肉/山药)、易消化 | 吃生萝卜 -> 提示：破气，少吃 |
| **阴虚质** | 忌辛辣、烧烤、羊肉、姜蒜 | 宜滋阴、银耳、百合、梨 | 吃烧烤 -> 警告：伤阴助火，建议吃梨润燥 |
| **血瘀质** | 忌寒凉、冷饮、油腻 | 宜活血、山楂、醋、泡脚 | 喝冷饮 -> 警告：寒凝血瘀，建议泡脚 |
| **气郁质** | 忌刺激(浓茶/咖啡/烈酒) | 宜疏肝、花茶、柑橘类 | 喝浓茶 -> 提示：可能加重焦虑 |
| **特禀质** | 忌发物(海鲜/鹅/笋/酒) | 宜清淡、固表、观察过敏 | 吃海鲜 -> 提示：注意观察皮肤反应 |
| **平和质** | 忌过量、油腻、高糖 | 宜均衡、多样化 | 正常饮食 -> 鼓励：保持即可 |

### 2.2. 智能睡眠管理 (Smart Sleep)
- **多模态记录**: 支持手动输入（“昨晚11点睡7点起”）或设备同步（Mock）。
- **中医时钟分析**: 
  - **子时 (23-1点)**: 胆经当令，熬夜伤胆气（口苦/决断力差）。
  - **丑时 (1-3点)**: 肝经当令，易醒代表肝火旺/肝血虚。
  - **寅时 (3-5点)**: 肺经当令，早醒代表肺气不足。
- **反馈**: 针对入睡时间与易醒时段，推送穴位按摩或食疗建议。

### 2.3. 健康周报 (Weekly Report)
- **生成机制**: 每周一自动生成上周总结卡片。
- **核心指标**: 
  - 睡眠达标率 / 饮食规律分 / 运动频次 / 排便状态。
- **AI 洞察**: 
  - 关联分析（Correlation）: “监测到您周二熬夜，周三记录了‘精神不振’，建议今晚早睡补救。”
  - 趋势预警: “连续 3 天摄入高糖，体质有转湿热风险。”

### 2.4. 快捷指令系统 (Shortcut System)
App 底部“+”号面板提供全局快捷操作，点击按钮直接触发对应预设卡片：

| 按钮名称 | 图标 | 触发指令 (Hidden Prompt) | 对应功能卡片 |
| :--- | :--- | :--- | :--- |
| **今日状态** | Activity (蓝) | `记录今日身体状态` | 状态选择器 (疲劳/焦虑/疼痛等) |
| **记饮食** | Utensils (橙) | `我要记饮食` | 饮食输入框 (支持语音/拍照) |
| **记睡眠** | Moon (紫) | `我要记睡眠` | 睡眠时间轴 + 质量评分 |
| **记排便** | PlusCircle (绿) | `我要记排便` | 布里斯托大便分类选择器 |
| **记月经** | Calendar (红) | `我要记经期` | 经期状态记录 (仅女性可见) |

### 2.5. 对话意图识别逻辑 (Intent Recognition Logic)
系统采用 **“混合识别模式 (Hybrid Mode)”**，结合本地正则与大模型语义理解，确保高精度识别。

1.  **分层识别策略**:
    *   **Level 0: 极速指令 (Fast Regex)**: 针对明确的快捷指令（如“我要记饮食”、“#睡眠”）进行毫秒级响应，零延迟唤起对应卡片。
    *   **Level 1: AI 意图分类 (LLM Intent)**: 对自然语言输入（如“刚吃了一碗牛肉面，感觉有点撑”）调用 LLM 进行意图分类，精准区分“记录”、“咨询”与“闲聊”。

2.  **核心意图定义**:
    *   **记录模式 (Record Mode)**: 
        *   触发条件：用户明确表达“发生了某事”（如“吃了...”、“睡了...”）或点击快捷按钮。
        *   行为：解析实体（食物/时间），生成 Daily Log，并给出针对性反馈。
    *   **咨询模式 (Inquiry Mode)**:
        *   触发条件：包含疑问词（“能吃吗”、“热量多少”）或“想吃...”。
        *   行为：调用 `analyze_diet` (Inquiry) 或通用 Chat，不生成 Log。
    *   **闲聊模式 (Chit-Chat)**:
        *   触发条件：无法归类为上述两种模式的通用对话（如“你好”、“天气不错”）。
        *   行为：进行富有同理心的对话。

3.  **打断与流转逻辑 (Interrupt Logic)**:
    *   **自动收起**: 当用户在某个记录卡片（如睡眠记录）未提交时，如果发送了新的“闲聊”或“其他记录”指令，系统将**自动放弃**当前卡片，转入新的对话流，防止卡片滞留阻碍交互。
    *   **全局复位**: 任何时候输入 `/reset` 或点击底部快捷按钮，均可强制结束当前流程，回到主对话状态。

### 2.6. 系统边界与模块关系 (System Boundaries)
**合拍 (Hepai)** 与 **藏真 (Cangzhen)** 是两个**完全独立**的业务模块，仅在底层共享用户账户体系。

*   **合拍 (Hepai)**: 
    *   定位：AI 健康管家（理性的左脑）。
    *   核心数据：User Profile, Daily Logs (Diet/Sleep/Poop)。
    *   入口：主对话界面。
*   **藏真 (Cangzhen)**: 
    *   定位：数字精神家园（感性的右脑）。
    *   核心数据：Memories (Memory Cards), Hall Layouts。
    *   入口：独立的“藏真”Tab 或页面。
*   **隔离原则**: 
    *   合拍的饮食记录**绝不**会出现在藏真的回忆墙上。
    *   藏真的心情日记**绝不**会影响合拍的体质分析（除非用户显式同步）。

## 3. 数据架构 (Data Schema)

### 3.1. 用户档案 (`userProfile`)
```json
{
  "basicInfo": {
    "gender": "female", // 'female' | 'male'
    "age": 25,
    "height": 165, // cm
    "weight": 50,  // kg
    "activity": "light", // 'light' | 'medium' | 'heavy'
    "constitution": {
      "type": "阳虚质",
      "desc": "阳气不足，失于温煦..."
    }
  },
  "settings": {
    "reminders": {
      "morning": "08:00",
      "night": "22:00"
    }
  }
}
```

以日期为 Key 的稀疏数组：
```json
{
```

## 4. 界面设计 (UI Design)
- **风格**: 极简、现代、高效。
- **主色调**: 
  - 品牌色: 绿色 (Emerald-500) 代表健康。
  - 辅助色: 橙色 (饮食)、靛蓝 (睡眠)、玫瑰 (经期)。
- **核心页面**:
  - **Chat Home**: 所有交互的发生地。
  - **Dashboard**: 顶部折叠面板，展示“今日数据”与“周报”。
  - **Profile**: 个人信息与设置。

## 5. 技术栈 (Tech Stack)
- **前端**: React 18, Vite, Tailwind CSS.
- **图标**: Lucide React.
- **状态管理**: React Context + localStorage (Persistence).
- **AI**: 
  - NLP: Doubao (Volcengine Ark).
  - Vision: Zhipu GLM-4V.
