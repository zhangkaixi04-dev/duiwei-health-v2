## 待开发功能 (按优先级排序)

### 📋 P0 高优先级

#### [ ] Task 1: 饮食记录双路径 - 路径A: 美食推荐卡片采纳
**目标**: 用户在「每日推荐」页面看到 AI 推荐的某餐食，点击「采纳」按钮直接记录。

**实现要点**:
1. 在 `DailyFeed.jsx` 的美食推荐卡片上添加「采纳」按钮
2. 点击后系统将该餐食的标准份量与营养数据写入 dailyLogs 和 nutrition_intakes
3. 前端实时更新「今日营养概览」卡片
4. AI 在聊天界面发送确认气泡："👌 早餐已记录：小米山药粥。开启元气满满的一天吧！"

**涉及文件**:
- `src/components/DailyFeed.jsx`
- `src/components/ChatInterface.jsx`

---

#### [ ] Task 2: 饮食记录双路径 - 路径B: 聊天框自然语言输入 + 语义模型分流
**目标**: 用户在聊天输入框中直接输入饮食内容，系统智能识别意图。

**实现要点**:
1. 添加意图分类逻辑（使用本地规则或模拟 LLM）
2. 意图分类 Prompt:
   ```
   你是一个意图分类机器人。请判断用户输入的内容属于以下哪一类：
   1. record_intent: 明确的饮食记录意图，包含食物名称。例如："我中午吃了麻辣香锅"、"记录一下，一杯拿铁"。
   2. chat_intent: 闲聊、提问或模糊的描述。例如："麻辣香锅好吃吗？"、"晚上吃什么好呢？"、"我今天吃得好饱"。
   
   请仅输出 "record_intent" 或 "chat_intent"。
   ```
3. 交互分支:
   - **识别为 record_intent 且无份量信息**: 弹出「份量追问卡片」
   - **识别为 record_intent 且有份量信息**: 直接解析并写入记录
   - **识别为 chat_intent**: 交由闲聊/问答模型处理

**涉及文件**:
- `src/components/ChatInterface.jsx`
- 新建: `src/services/intentService.js` (意图分类服务)

---

#### [ ] Task 3: 份量追问卡片 (Portion Prompt Card)
**目标**: 当识别为 record_intent 但缺少份量信息时，弹出卡片让用户选择。

**卡片规格**:
- 标题: "大约吃了多少呢？"
- 选项 (图文并茂):
  - [Icon] 50g (一小把 / 约几口)
  - [Icon] 100g (约手掌大小)
  - [Icon] 150g (正常一人份)
  - [Icon] 200g (份量超足)
- 底部按钮: "自己输入克数" (文字按钮)
- 交互: 用户点击任一选项后，卡片消失，系统将选择的份量与食物一同写入记录，并发送确认气泡

**涉及文件**:
- 新建: `src/components/PortionPromptCard.jsx`
- `src/components/ChatInterface.jsx`

#### [ ] Task 4: 经期记录关键词触发
**目标**: 将经期记录功能改为通过自然语言触发，简化主界面。

**触发关键词**:
- 来姨妈了, 大姨妈, 月经, 生理期, MC

**交互流程**:
1. 用户在聊天框输入任一触发词
2. AI 首先发送引导气泡："请在下方卡片记录您的经期情况，我会为你同步调整后续的饮食和运动建议。📅"
3. 系统立即弹出「经期记录卡片」

**卡片规格**:
- 主题色: 玫红色 (#f43f5e)
- 四维度记录:
  - 经血量: 少量 / 中量 / 多量 (每个选项下方有简短说明)
  - 颜色: 鲜红色 / 暗红色 / 褐色 / 淡粉色 (每个选项下方有简要含义说明)
  - 痛感: 无 / 轻微 / 中等 / 严重 (每个选项下方有描述)
  - 心情: 😄 开心 / 🙂 正常 / 😰 焦虑 / 😤 烦躁 / 😔 低落
- 底部按钮: [确认记录]，点击后将数据写入 menstrual_logs 表，并回复确认气泡

**涉及文件**:
- `src/components/ChatInterface.jsx` (关键词检测 + 消息处理)
- 新建/修改: `src/components/PeriodRecordCard.jsx`
