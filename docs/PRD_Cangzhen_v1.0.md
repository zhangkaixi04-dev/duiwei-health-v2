# 藏真APP - 产品需求文档（PRD）V1.0

## 一、文档基础信息

- **产品名称**：藏真
- **文档版本**：V1.0
- **核心目标**：打造「个人真实体验线上博物馆」，对抗数字沉迷，引导用户记录、沉淀、回顾真实生活；以“莫兰迪色系+玻璃拟态玻璃花房”为视觉核心，强化“珍藏即种花”的治愈仪式感，实现轻量化、有温度的内容记录与回顾
- **适用场景**：用户记录生命瞬间、情绪感受、灵感碎片、五感体验，回顾过往珍藏内容，以及V2.0阶段与亲友共建共同回忆

## 二、产品背景与立项理由

### 2.1 市场痛点

- **数字沉迷与真实感知脱节**：用户被碎片化信息流裹挟，过度关注虚拟内容，忽视对现实世界的五感体验、情绪流动与灵感捕捉。
- **回忆记录碎片化且无沉淀**：现有记录工具（备忘录、相册）仅做存储，无分类、无仪式感、无深度回顾，珍贵回忆沦为 “数据垃圾”。
- **回顾体验缺乏情感温度**：多数总结类功能偏向数据化统计，缺少人文感、诗意感，无法满足用户对自我成长与情感沉淀的需求。
- **深度关系回忆无专属载体**：亲友间的共同回忆分散在社交软件，无专属空间进行集中珍藏与双向回溯，情感连接缺乏具象化载体。

### 2.2 产品价值

- **用户价值**：提供 “记录 - 分类 - 回顾 - 共建” 的全链路服务，让用户重新聚焦真实生活，沉淀个人成长轨迹，构建有温度的回忆体系；以玻璃花房场馆+花朵动效，赋予记录与回顾仪式感，强化治愈体验。
- **产品价值**：以「莫兰迪玻璃花房博物馆」为核心体验，打造差异化回忆记录产品，填补 “真实感知 + 诗意回顾 + 双向共建 + 治愈视觉” 的市场空白。

## 三、产品定位

### 3.1 核心定位

藏真是一款以「莫兰迪玻璃花房线上博物馆」为载体，通过引导用户手动分类记录真实事件、沉淀回忆、生成诗意周总结，并支持好友共建专属展馆的轻治愈型生活记录产品。

### 3.2 目标用户

- **核心用户**：20-40 岁，注重生活品质、情感沉淀，受数字沉迷困扰，希望记录真实生活、梳理自我成长，偏爱温柔治愈视觉风格的都市人群。
- **潜在用户**：重视深度关系，希望与亲友共同珍藏回忆，喜欢仪式感记录方式的用户。

### 3.3 产品边界

- **做**：真实生活体验记录、手动分类沉淀、诗意周总结、个人 / 双人博物馆回溯、藏品数量统计、玻璃花房场馆体验、花朵仪式感动效。
- **不做**：社交广场、内容变现、复杂的图文编辑、游戏化激励体系、高饱和视觉设计、花哨动效、AI自动分类功能。

## 五、核心视觉与交互规范（必遵循）

### 5.0 性能与适配原则
- **机型策略**：优先保障高端机型（旗舰芯片）体验，不针对中低端机型进行降级适配。确保玻璃拟态、实时模糊与粒子动效的极致流畅。
- **权限引导**：在首次启动或使用对应功能时，必须弹出莫兰迪风格的引导弹窗，明确告知“为了记录真实生活，需要访问相机/麦克风/相册”。
- **存储策略**：
    - 仅存储文字与**压缩后的图片/音频**。
    - 高清原图/无损音频不上传云端，仅本地暂存或压缩后上传，降低云存储成本。
    - **云端备份**：支持手机号登录后，自动同步“压缩后的珍藏”至云端，确保换机不丢失。

### 5.1 整体风格定义

- **主风格**：莫兰迪低饱和色系 + 玻璃拟态（Glassmorphism），贴合“纯白博物馆”核心定位，主打“朦胧、通透、温柔、治愈”，无尖锐棱角、无高饱和色彩冲击，保留充足留白与纸张质感。
- **核心质感**：磨砂玻璃（模糊度可控）、轻微高光白边、低透明度底色，实现“玻璃通透感+莫兰迪柔和感+博物馆纸张感”三重叠加，不破坏纯白基调。
- **视觉层级**：背景（莫兰迪米白+朦胧花房纹理）→ 玻璃容器（场馆/卡片）→ 内容（文字/花朵动效）→ 交互元素（玻璃按钮），层级清晰不杂乱，突出内容本身。
- **字体规范**：标题（楷体，柔和有温度，字号18-24sp）；正文（宋体，清晰干净，字号14-16sp）；统计文字（极简黑体，字号16-20sp）；行高≥1.5，确保阅读舒适度，贴合整体温柔调性。

### 5.3 玻璃拟态统一规范

- **所有玻璃元素**（场馆卡片、按钮、输入框、藏品卡片）必须满足：磨砂模糊效果 + 1px白色高光边（顶部/左侧）+ 1px浅灰阴影（底部/右侧），营造立体玻璃质感，阴影低透明度（≤20%），贴合博物馆柔和氛围。
- **圆角规范**：所有玻璃元素统一圆角20px（无直角设计），藏品卡片圆角16sp，按钮圆角12sp，贴合花房柔和感，避免尖锐感，与整体极简风格统一。
- **透明度规范**：容器类（场馆、卡片）透明度60%-70%，交互类（按钮）透明度70%-80%，输入框透明度80%，确保背景朦胧透出，同时不影响内容可读性，不破坏纯白基调。
- **适配要求**：玻璃效果在浅色背景下正常显示，深色模式（若有）同步降低模糊度至5-8px，保持通透感不丢失；适配全面屏、刘海屏、折叠屏，玻璃花房形态不畸变。

### 5.4 动效设计规范（核心）

- **核心原则**：轻盈、安静、有仪式感，无花哨动效，服务于“博物馆+玻璃花房”体验，动效时长200-300ms，缓动函数采用ease-out/ease-in-out，确保流畅无卡顿。
- **关键动效（MVP阶段）**：
    - **藏品入馆**：柔和白光闪烁（150ms）→ 对应场馆主题小花缓缓绽放（300ms）→ 藏品卡片轻盈缩小，平滑飞入对应玻璃花房（300ms）→ 顶部轻提示“藏品已入XX馆”（带小花图标）。
    - **场馆进入**：点击分馆入口→ 页面淡入过渡（200ms）→ 对应场馆主题花完整盛开动效（500ms）→ 展厅页面完全展示，顶部显示「XX馆・共X件藏品」。
    - **逛展滑动**：左右滑动屏幕切换藏品，核心藏品居中显示（尺寸最大、清晰度最高），两侧藏品轻微缩小（80%）、透明度降低（60%），模拟视线聚焦；滑动时卡片平滑过渡，无回弹、无多余动效，玻璃卡片带轻微位移模糊。
    - **驻足查看**：点击核心藏品→ 藏品卡片缓慢放大至全屏（300ms），背景保持莫兰迪米白，内容（文字/图片/音频）清晰展示；音频自动播放（静音状态下除外）；退出时卡片缓慢缩小，回到展厅原位（300ms），保持当前浏览进度。
    - **按钮交互**：点击玻璃按钮→ 轻微放大（1.05倍）+ 亮度提高（10%）+ 玻璃反光动效（200ms）；松手后恢复原状，无多余反馈。

**V2.0共建馆动效**：
    - **双向记录**：两道浅蓝光影（对应双方）汇聚成一张藏品卡片（300ms），象征“连接”，卡片生成后附带双小花图标（双方主题花）。
    - **双人在线**：屏幕显示两个半透明光标（对应双方），光标移动时带轻微玻璃反光；查看同一藏品时，藏品卡片边缘浮现双主题花环绕动效（200ms）。

### 5.6 交互仪式感与书写激励 (New)

为了提升用户的“书写欲望”，将“填表”这一枯燥行为转化为“策展”的神圣仪式。

#### 5.6.1 视觉质感：从平面到“物质感” (Materiality)
- **纸张纹理 (Canvas)**：输入区域不再是纯白背景，而是叠加一层极淡的**粗颗粒水彩纸**或**和纸 (Washi)** 纹理。让用户感觉是在一张有厚度的卡片上书写。
- **动态光影 (Lighting)**：引入随陀螺仪微动的**动态光影**。早晨呈现清透侧光（希望感），深夜呈现温暖顶光（聚拢感，类似博物馆射灯）。
- **玻璃呼吸 (Respiration)**：未填写的卡片呈现“半透明磨砂”状态（未擦拭的窗）；随着文字输入，卡片逐渐变得“晶莹剔透”（被擦亮的玻璃），隐喻“记录让记忆变得清晰”。

#### 5.6.2 交互流程：从提交到“封存” (The Ritual)
- **入场：拾笔 (Pick up)**：首页底部入口设计为一支**微微晃动的悬浮笔**或**空展位**。点击后，镜头**Zoom-in 拉近**，仿佛用户走到了展柜前，玻璃罩缓缓升起。
- **过程：沉浸 (Flow)**：开始输入时，周围 UI（顶栏/底栏）全部后退并模糊（Blur），只留下当前卡片。文字上屏带有**墨水晕染**的微动效。
- **终局：封存 (Seal)**：
    - **拒绝“提交”**：按钮文案改为**“封存”**或**“入馆”**。
    - **动作设计**：点击后，播放**“盖章”动画**（生成日期印戳），随后玻璃罩**缓缓落下**（伴随清脆的 *咔哒* 声），卡片被自动推入后方长廊。

#### 5.6.3 感官反馈 (Sensory)
- **听觉设计**：
    - **白噪音**：进入记录页时，背景淡入极轻微的**博物馆环境音**（空灵脚步声/远处雨声）。
    - **书写音**：键盘音效替换为**钢笔划过纸张的沙沙声**（ASMR级别），每打一个字都是一种享受。
- **触觉反馈**：入馆封存瞬间，给予一次**沉稳、短促的震动** (Heavy Impact)，确认感极强。

#### 5.6.4 引导策略：把填空题变成“问答题”
输入框默认文案采用**“每日灵感提问”**，而非“请输入内容”。
- *示例*：“今天那一刻的风，是什么温度？” / “刚刚吃的那口甜点，让你想起了谁？” / “此刻的你，想对明天的自己说什么？”

## 六、核心功能需求（MVP：V1.0）

### 模块 1：首页（核心入口）

#### 6.1.1 功能概述
作为产品总入口，聚合核心操作与关键内容，保持莫兰迪米白+玻璃拟态风格，突出 “记录” 与 “回溯” 核心路径，大量留白，贴合博物馆治愈氛围。

| **个人博物馆入口** | 醒目标识「我的玻璃花房博物馆」，附带“个人/共建双馆”提示（玻璃小图标），玻璃按钮样式。 | 点击 → 进入博物馆主页面，双Tab切换栏与总馆玻璃花房元素缓慢淡入（300ms），兰花轮廓轻微舒展（500ms），默认展示个人博物馆Tab内容。 |

### 模块 2：记录页（内容输入+手动分类）

#### 6.2.1 功能概述
支持用户完成真实事件的记录与手动分类，无次数限制，操作极简，玻璃拟态输入界面，完成分类提交后触发 “藏品入馆” 仪式感动效（主题花绽放+卡片入馆），分类准确可后续调整，提升用户操作便捷性。

#### 6.2.2 核心功能与交互

| 功能 | 描述 | 交互逻辑 |
| :--- | :--- | :--- |
| **多形式记录** | 支持文字（无字数限制）、图片（1条记录仅支持单张），玻璃形态输入框（莫兰迪米白底色，8px模糊）。 | 1. 文字：直接输入，自动换行，字体为宋体16sp；2. 图片：点击玻璃相机图标，选择拍摄/相册，预览图为玻璃卡片样式；3. 音频：点击玻璃麦克风图标，长按录制，松手结束，录制进度条为对应场馆辅助色。 |
| **主题自定义** | 自动填充引导事件主题（若从引导入口进入），支持用户手动修改，主题栏为玻璃形态。 | 点击主题栏 → 编辑文字，确认后保存，主题栏边框高亮（对应场馆辅助色），无多余动效。 |

### 模块 3：分类修正与不可逆原则

#### 6.3.1 功能概述
用户手动分类后，若发现分类不准确，可在藏品详情页或博物馆内手动调整分类。
**重要限制**：一旦藏品“入馆”（点击确认存入），即视为永久珍藏。
- **不可编辑**：内容（文字/图片/音频）不可修改，保留当时的真实状态。
- **不可删除**：藏品不可删除，仅可“雪藏”（在列表中隐藏，但在总数统计中保留），强化“落子无悔”的真实感。

#### 6.3.2 四大分馆分类标准 (MECE原则优化)

为了避免“感知”与“情绪”的重叠，并符合 MECE 原则（相互独立，完全穷尽），我们将四大分馆重新定义为 **“外在体验 - 内在反应 - 抽象思考 - 实体存在”** 的闭环：

| 场馆名称 | 核心定义 (Key Definition) | 包含内容 (Scope) | 典型示例 (Examples) |
| :--- | :--- | :--- | :--- |
| **感知馆 (Sensation)** | **向外的触角**。纯粹的物理感官输入，不带主观评价。关注“世界是什么样的”。 | 视觉、听觉、嗅觉、味觉、触觉的直接记录。 | 晚霞的颜色、雨滴的声音、咖啡的香气、猫咪的绒毛触感。 |
| **情绪馆 (Emotion)** | **向内的回响**。对外界刺激的主观心理反应。关注“我感觉怎么样”。 | 喜怒哀乐、焦虑、平静、感动、遗憾等心理状态。 | “今天莫名很低落”、“听到这首歌突然想哭”、“被陌生人的善意暖到了”。 |
| **灵感馆 (Cognition)** | **大脑的产出**。经过思考后的观点、顿悟或创意。关注“我想到了什么”。 | 书摘、影评、突发的脑洞、对某件事的深刻反思、梦境。 | “突然明白了那句话的含义”、“想到了一个绝妙的文案”、“昨晚梦见自己在飞”。 |
| **人间馆 (Life)** | **存在的痕迹**。具体的、有实体的生活切片。关注“我经历了什么”。 | 美食、物件、动植物、具体的事件（如旅行、聚会）。 | 一顿丰盛的晚餐、新买的杯子、路边的一朵花、一次长途飞行。 |

*(注：原“生命馆”更名为“人间馆”，涵盖范围更广，且更有烟火气。)*

#### 6.3.3 分类修正交互

- **路径 1（藏品详情页）**：点击「修改展馆」玻璃按钮 → 弹出四大馆玻璃选项（带主题花图标+分类提示）→ 选择目标展馆 → 提示 “藏品已移至 XX 馆”（带主题花动效），藏品卡片同步切换为目标场馆玻璃质感。
- **路径 2（博物馆内）**：长按藏品卡片（玻璃卡片轻微上浮1px，200ms）→ 拖拽至目标分馆入口（目标分馆玻璃花房轻微高亮）→ 松手完成移动（动效：卡片平滑移动至对应展馆，300ms，附带主题花飘落动效）。

### 模块 4：个人博物馆（核心体验）

#### 6.4.1 功能概述
以「莫兰迪玻璃花房博物馆」为视觉基调，实现 “漫步逛展” 式的回忆回溯，展示四大分馆的所有藏品及统计数据，强化治愈仪式感。

#### 6.4.2 页面层级

- **博物馆主页面**：莫兰迪米白背景，顶部设置玻璃质感双Tab切换栏（圆角20px，模糊8px，透明度70%），Tab1为「个人博物馆」（默认选中，带兰花图标，高亮显示），Tab2为「共建博物馆」（V2.0上线后显示，带双花组合图标，默认未选中），Tab切换时附带轻微玻璃反光动效（200ms），页面内容平滑淡入淡出。
- **个人博物馆Tab（默认展示）**：展示四大玻璃花房分馆入口（带各自藏品数量、主题花图标），以“展柜标识牌”形式排列，大量留白，玻璃花房轻微悬浮质感；顶部显示「个人博物馆・共X件藏品」标题，带兰花图标。
- **共建博物馆Tab（V2.0上线后显示）**：展示用户创建/加入的共建馆列表（玻璃卡片样式，带双花图标、共建天数及总藏品数），顶部显示「共建博物馆・共X个展馆」标题，支持点击进入对应共建馆，延续玻璃花房视觉风格。

### 模块 5：AI 回顾与总结（新增核心）

#### 6.5.1 功能目标
利用 AI 能力，对用户的碎片化记录进行深度语义分析，生成具有人文关怀的**周总结、月总结与年度总结**。
- **核心指标**：内容广度（覆盖多维度记录）、人文关怀（治愈系文案）。

#### 6.5.2 生成策略（确保广度与关怀）

**A. 广度保障策略 (Recall Strategy)**
为了避免 AI 只关注最近几条记录，采用 **“全量标签 + 关键帧采样”** 机制：
1. **预处理 (Tagging)**：每条记录入馆时，AI 自动提取 3-5 个关键词（如“雨天”、“焦虑”、“咖啡”），存入向量数据库。
2. **采样逻辑**：
   - **高频词覆盖**：选取本周期内出现频率最高的 3 个情绪词。
   - **冷门惊喜**：随机选取 1 条被遗忘的“小确幸”（低频但情感正向）。
   - **时间跨度**：强制包含周期初、中、末的记录，形成完整叙事线。
3. **多模态理解**：对图片进行场景识别（如“日落”、“猫咪”），作为辅助上下文输入 AI。

**B. 峰终定律与金句库 (Peak-End & Copywriting)**
借鉴小红书的“人文关怀”逻辑，AI 生成总结时需严格遵循**峰终定律**，并调用**分层金句库**：

1. **结构化生成 (Template Structure)**：
   - **开头 (The Start)**：**破冰与共情**。不展示冷冰冰的日期，而是用季节、天气或整体氛围切入。
     - *金句库示例*：“这一周的雨水很多，但你的心里似乎藏着一个小太阳。” / “在忙碌的缝隙里，你依然捕捉到了 5 次心动的瞬间。”
   - **中间 (The Peak)**：**高光时刻重现**。选取本周期内**情感浓度最高**（Tag 关联度最强）或**互动最深**（驻足时长最长）的 1-2 条记录，进行深度演绎。
     - *策略*：AI 必须引用原文细节，并进行升华。例如用户记录了“加班后的关东煮”，AI 升华为“那碗热汤，是深夜里最温柔的抚慰”。
   - **结尾 (The End)**：**升华与留白**。给出一个具有“获得感”的结论，并留下对下一周期的期待。
     - *金句库示例*：“万物皆有裂痕，那是光照进来的地方。” / “收藏好这份温柔，我们要继续赶路了。”

2. **去数据化表达 (De-datafication)**：
   - 禁止出现“你本周记录了 12 条”这种机械描述。
   - 转化为“你与 12 个真实瞬间相遇”、“你留住了 12 份易逝的感动”。

**C. 人文关怀策略 (Tone & Persona)**
通过 Prompt Engineering 塑造 AI 的 **“博物馆馆长”** 或 **“时光园丁”** 人格：
- **角色设定**：一位温柔、敏锐、不评判的观察者，用诗意的语言串联碎片。
- **文风要求**：莫兰迪色系的文字风格——平和、细腻、治愈。

#### 6.5.3 交互形式
- **周总结**：每周日晚 20:00 推送，生成一张“玻璃信笺”，包含一段 AI 寄语 + 3 张精选藏品回顾。
- **月总结**：生成一本“月度花册”，包含情绪曲线（以花朵盛开程度表示）+ 关键事件回顾。
- **年度总结**：生成“年度博物馆导览视频”（动态生成的 H5/视频），按时间轴重现这一年的心路历程。
